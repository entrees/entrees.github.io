Chapter 5
Clause-within-clause subordination


5.1   Introduction

The hierarchy of layers from chapter 1 starts with a matrix clause as the highest layer with phrases subsequently reached. The picture of internal clause structure was fleshed out further in chapter 4 to include clause intermediate layers (IML) with coordination. The hierarchy from chapter 1 also allows two types of clause subordination:

  • clause-within-phrase, and
  • clause-within-clause.

Clause-within-phrase possibilities were seen in chapter 3 with discussions of modifiers and complements inside phrases. This chapter details clause-within-clause options. In chapter 4, clause elements were realised by (noun) phrases with subject, object, predicative, and adverbial roles. Now we will see how such elements can also be clauses themselves. The discussion will group subordinate clauses by the node labels of their maximal constituent:

  • IP-INF (section 5.2),
  • IP-PPL (section 5.3),
  • CP-THT (section 5.4),
  • CP-QUE (section 5.5), and
  • PP-SCON (section 5.6).

With the exception of CP-QUE, which is used as the tag for both direct and indirect questions, the other tags are used to label clauses that are incapable of standing alone as the main clause of a sentence. When they are all the content of an utterance, they are inside a FRAG layer.


5.2   IP-INF (infinitival clause)

Infinitival clauses are clauses that contain a verb in the infinitive form without support from a finite element like a dummy DO or modal verb (as seen with matrix clauses in section 4.2.1). Infinitival clauses have a maximal constituent with the tag of Table 5.1.

Table 5.1: Tag for infinitival clause

IP-INFinfinitival clause

There are two kinds of infinitival clause:

  • those that contain the infinitive marker to (tagged TO), typically as the clause initial word, and
  • those that lack the infinitive marker to, called bare infinitival clauses.

When occuring directly within a clause, the syntactic tag IP-INF needs a function extension. Possibilities are listed in Table 5.2.

Table 5.2: Tags for infinitival clauses with function extensions

IP-INF-CATcatenative
IP-INF-(N)SBJ(notional) subject
IP-INF-(N)OB1(notional) direct object
IP-INF-PRD2subject predicative
IP-INF-PRDobject predicative

Chapter 6 will show infinitival clauses as verb selected elements with direct object, object predicative, and catenative functions. Notably, instances of bare infinitival clauses occur as:

  • bare IP-INF-OB1 with code [Tni] (section 6.2.3.11),
  • bare IP-INF-PRD with code [Cn.i] (section 6.2.4.7), and
  • bare IP-INF-CAT with code [cat_i] (section sectionref:[__cat_ipattern__]).

Infinitival clauses with the infinitive marker to exhibit a fuller range of options, occurring as:

  • IP-INF-OB1 with code [It] (section 6.2.2.5).
  • IP-INF-OB1 with code [Tt] (section 6.2.3.6).
  • IP-INF-OB1 with code [Tnt] (section 6.2.3.7).
  • IP-INF-PRD with code [Cn.t] (section 6.2.4.5),
  • IP-INF-OB1 with code [Dn.t] (section 6.2.4.14),
  • IP-INF-OB1 with code [Dpr.t] (section 6.2.4.15), and
  • IP-INF-CAT with code [cat_t] (section sectionref:[__cat_tpattern__]).

The remainder of this section notes how infinitival clauses with the infinitive marker to can occur as subjects (section 5.2.1) and subject predicatives (section 5.2.2).

5.3   IP-PPL (participial clause)

Participial clauses have a maximal constituent with the tag of Table 5.3.

Table 5.3: Tag for participial clause

There are two kinds of participial clause:

  • those that contain a past participle ({en}) verb form, and
  • those that contain a present participial ({ing}) verb form.

When occuring directly within a clause, the syntactic tag IP-PPL needs a function extension. Possibilities are listed in Table 5.4.

Table 5.4: Tags for participial clauses with function extensions

Chapter 6 will show participial clauses as verb selected elements with direct object, object predicative, and catenative functions. Notably, instances of participial clauses with a past participle ({en}) verb form occur as:

  • IP-PPL-PRD with codes [VP24A, VP24B, VP24C] (section 6.3), and
  • IP-PPL-CAT with code [cat_e] (section sectionref:[__cat_epattern__]).

  • IP-PPL-OB1 with code [Tg] (section 6.2.3.8),

5.4   CP-THT (that clause)

A that-clause is an embedded clause with a finite verb. This may have the complementizer that (tagged C) as the clause initial word. A that clause has a maximal constituent with the tag of Table 5.5 and, when structurally complete, contains a finite clause layer (IP-SUB) which in turn contains all the clause content, including any clause initial complementizer.

Table 5.5: Tag for that-clause

CP-THTTHAT clause

When occuring directly within a clause, the syntactic tag CP-THT needs a function extension. Possibilities are listed in Table 5.6.

Table 5.6: Tags for that-clauses with function extensions

CP-THT-(N)SBJ(notional) subject
CP-THT-(N)OB1(notional) direct object
CP-THT-PRD2subject predicative

Chapter 6 will show THAT clauses as verb selected elements with direct object function:

  • CP-THT-OB1 with code [Dpr.f] (section 6.2.4.11).

This section considers clauses that are the subordinate component of subordinate conjunctions. These clauses can differ with regards to:
